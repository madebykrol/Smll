Membership framework

Smll comes packed with a internal framework for managing members in your application, all the way from registration, to activtion to validation, to editing and deletion of members. This is done through the interfaces IMembershipProvider and IRoleProvider

IMembershipProvider

The IMembershipProvider interface mananges your users, the only current implementation of this interface is SqlMembershipProvider which manages membership data in your database.

Create a user

IMembershipProvider has a createUser method that takes 3 parameters, 2 required and one optional.

ParameterDescription
UsernameThe username of the new member
PasswordA password for the user, this password will be hashed and stored by the provider
ProviderUserKeyOptional. The key, associated with this user, default is a Guid
When this method is invoked it will try to pass this information as a new user, and will store this information in the membership table. if the user exists it will throw a UserExistsException

Validating a user

Validating a user is commonly done in conjuction with the IAuthenticationProvider to validate a users credentials before authenticating it.
For this purpose IMembershipProvider has a method called validateUser which takes 2 parameters.

ParameterDescription
UsernameThe username
PasswordThe password for the user, will be hashed and matched against the stored password for the user
If the user does not exist, validateUser will throw a UserDoesNotExistException

Enabaling / Disabaling a user

IAuthenticationProvider

Together with IMembershipProvider this interface makes out one very powerfull feature in Smll, the memmbership framework. this interface handles authentication of memberships by creating a AuthenticationTicket that is being validated on each request to keep a user authenticated. The default implementation of IAuthenticationProvider is FormAuthentication

Setting the authentication ticket / Signing in!

By calling the method signin with a validated username IAuthenticationProvider will authenticate the user by setting a AuthenticationTicket to the response. This user must then carry this ticket on each consecutive request to stay signed in.

ParameterDescription
UsernameA validated username.
updateLastLoginDateOptional. if true, IAuthenticationProvider, will update the last login date for this user.

Removing the ticket / Signing out!

By calling signout the AuthenticationProvider will destroy the ticket, and invalidate the token, leaving the user unauthenticated untill he / she signs in again.

Checking if a user is authenticated

The IAuthenticationProvider has a getPrincipal method, which returns a object that implements the IPricipal. This interface lets you grab a Identity object through getIdentity which lets you test wheater or not the user of the current context is authenticated or not by invoking isAuthenticated method.

IRoleProvider

As any application might need different levels of authentication. Smll comes with a IRoleProvider interface. this interface provides means to handle membership roles, such as Administrators, Editors, and Users.

A list of potential roles

You can recieve a list of roles that this application is providing by invoking IRoleProviders method getRoles

ParameterDescription
None

Check if user is in a role

Using the findUserInRole method you get a boolean if the user is in a specific role

ParameterDescription
UsernameThe username you want to find roles for.
RoleRole to search in.

Get all roles of a user

getRolesForUser returns a list of all the roles that this user belongs to.

ParameterDescription
UsernameThe username you want to find roles for.

IRoleProvider together with IAuthenticationProvider

IAuthenticationProvider provides a IPrincipal that keeps track of the roles that the current user has. this makes it really easy to test weather the current user has a specific role, for instance if the user is a Administrator of your application. by calling the isInRole with Administrator as it's parameter you get a boolean telling you if the user is in this role or not.

ParameterDescription
RoleA string representing the role you want to test.
